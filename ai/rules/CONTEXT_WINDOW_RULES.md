# 🧠 RULE 03: AI CONTEXT WINDOW PROTOCOL

> **Primary Master Rule for `appended-context-window.html`**

---

## 🎯 Purpose

To maintain a persistent, visual context dashboard in `appended-context-window.html` for future AI subagents, sessions, and developers.

---

## 📜 Mandatory Operational Requirements

1. **Target File**: `appended-context-window.html` located at project root.
2. **Self-Contained Architecture**:
   - Must use **inline CSS** and **inline JavaScript**.
   - Memphis / Neo-Brutalist 90s visual styling matching system standards.
3. **Core Sections**:
   - **System State Dashboard**: Active mode, platform targets, active rules.
   - **Architecture & File Map**: Tree view of key files and modular structure.
   - **Current Sprint & Goals**: Active task list with status indicators.
   - **AI Memory Digest**: Summary of architectural decisions, past edits, and context notes for future agents.
4. **Maintenance Frequency**:
   - Update `appended-context-window.html` whenever major changes, structural additions, or strategy decisions are made.
